# ⭐ Introduction

- SACH-AI is a multimodal AI system designed to help people verify whether digital content is real, manipulated, or misleading.<br>
- The system analyzes text, images, videos, audio messages, and screenshots to detect deepfakes, scams, misinformation, and AI-generated media — and explains why the content may be suspicious.<br>
- Our goal is simple:
  "Build a trust layer for the internet that helps users verify information before it spreads."

# 🌍 Why SACH-AI?

Today the internet is flooded with:

- 🎭 Deepfake videos
- 🖼️ AI-generated images
- 💬 Fake viral messages
- 💳 Scam and phishing attempts
- 📰 Misleading or manipulated news

These can spread to millions of people in minutes, often causing real-world harm.<br>
Most users have no simple way to verify authenticity.<br>
SACH-AI solves this problem by combining multiple AI systems into one unified verification engine.

# 🚀 Key Features

- 🧠 Multimodal Analysis<br>
Analyzes text, images, videos, and audio in one system.

- 🔍 Deepfake Detection<br>
Detects manipulated videos and AI-generated faces.

- 💬 Fake Message Detection<br>
Identifies misleading messages and misinformation.

- 🖼️ Image Authenticity Check<br>
Detects AI-generated or manipulated images.

- 🎙️ Voice Analysis<br>
Detects AI-generated or cloned voices.

- 📊 Trust Score System<br>
Assigns authenticity scores based on analysis.

- 🤖 AI Explanation Engine<br>
Explains why the system believes content is suspicious.

# 🧠 System Architecture

SACH-AI works as a multimodal verification pipeline.<br>
Instead of one large model, the system uses specialized AI models for different types of content, and then combines their results.

Pipeline overview:

User Input<br>
↓<br>
Content Type Detection<br>
↓<br>
Multimodal Analysis<br>
↓<br>
Evidence Retrieval<br>
↓<br>
AI Explanation Engine<br>
↓<br>
Authenticity Score

### 📥 Step 1 — Content Input

Users can submit:<br>
- 🖼️ Images
- 🎥 Videos
- 🎙️ Audio messages
- 💬 Text messages
- 📱 Screenshots
  
The system first identifies what type of content it is.

### 🔎 Step 2 — Content Type Detection

SACH-AI determines the input type:<br>
- Image
- Video
- Audio
- Text
- Screenshot
  
Based on this, the system routes the content to the correct analysis pipeline.

### 🖼️ Step 3 — Image Analysis Pipeline

If the input is an image, SACH-AI performs several checks.<br>
###### 🔤 Text Detection
The system first checks if the image contains text.<br>
Tool used:<br>
- OCR (Optical Character Recognition)

Example:

Screenshot of viral message<br>
↓<br>
Extract text<br>
↓<br>
Send to text verification module

###### 🧠 AI Image Detection
The system checks whether the image was generated by AI.

Detection methods:

- GAN artifact detection
- texture anomaly detection
- neural image classifiers

###### 🛠️ Manipulation Detection
The system checks whether the image was edited.

Possible signs:
- copy-paste manipulation
- lighting inconsistencies
- abnormal pixel artifacts

### 🎥 Step 4 — Video Analysis Pipeline
Videos are analyzed through multiple steps.<br>
###### 📸 Frame Extraction
The video is broken into frames.

Video<br>
↓<br>
Extract frames<br>
↓<br>
Analyze faces and visuals

###### 🎭 Deepfake Detection
Each frame is analyzed for deepfake artifacts.

Possible indicators:

- facial blending errors
- unnatural blinking
- inconsistent skin textures

The system calculates a deepfake probability score.

###### 🔊 Audio Extraction
The system extracts audio from the video and sends it to the audio analysis pipeline.


### 🎙️ Step 5 — Audio Analysis Pipeline
If the input contains audio:
###### 🗣️ Speech-to-Text
Speech is converted to text.<br>
This allows the system to analyze claims made in the audio.<br>

###### 🎭 Voice Deepfake Detection
The system checks for:

- cloned voices
- AI generated speech
- unnatural voice patterns

### 💬 Step 6 — Text Verification Pipeline
Text can come from:

- typed messages
- extracted image text
- speech transcripts

The system performs:

###### 🚨 Scam Detection
Detects common scam patterns such as:

- urgent payment requests
- fake bank alerts
- phishing messages
  
###### 📰 Claim Verification
The system extracts factual claims.

Example:

- "Government giving ₹10,000 scheme"
- Then it verifies the claim using trusted knowledge sources.

### 🌐 Step 7 — Evidence Retrieval
SACH-AI uses Retrieval Augmented Generation (RAG).

###### Process:

Extract claim<br>
↓<br>
Search trusted sources<br>
↓<br>
Retrieve information<br>
↓<br>
Compare with input

###### Sources may include:

- news databases
- fact-checking sites
- verified knowledge sources

### 🤖 Step 8 — AI Explanation Engine

Instead of simply labeling content as fake or real, SACH-AI explains the reasoning.

- Example output:
 
```
Deepfake probability: 82%

Reasons:
- facial blending artifacts detected
- inconsistent frame textures
- no trusted sources confirm event

This makes the system transparent and understandable for users.
```

### 📊 Step 9 — Authenticity Score

The final system output includes:

```
- Authenticity Score: 24%
- Risk Level: HIGH
- Recommendation:
Do not share this content.
```
